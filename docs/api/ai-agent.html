AiAgent (sn_aia_agent)

Configure an AI Agent using the Fluent API. This API provides a structured interface for creating AI Agents in ServiceNow's AI Agent Studio, automatically handling all underlying complexity and relationships between multiple tables.

Overview​

The AiAgent API enables you to define intelligent agents that can:

  • Execute Tools: Use built-in or custom tools (CRUD, Script, RAG, Web Search, etc.)
  • Process Context: Apply custom scripts to transform context before execution
  • Control Access: Define role-based security and data access controls
  • Trigger Automatically: Configure triggers for automatic agent invocation
  • Manage Versions: Support multiple versions with different configurations

The plugin automatically generates all required ServiceNow records (~9 tables) with proper relationships and defaults.

Version Details (versionDetails)​

Version information for the agent. This is an array that can contain multiple versions.

interface AiAgentVersionDetailsType {
  name?: string;                 // Version name (e.g., "V1", "V2")
  number?: number;               // Version number (e.g., 1, 2)
  state?: AiAgentVersionState;   // Version state
  condition?: string;            // Condition for this version
  instructions?: string;         // Version-specific instructions that guide the agent's behavior
}

type AiAgentVersionState = 'draft' | 'committed' | 'published' | 'withdrawn';

Example:

versionDetails: [
  {
    name: 'V1',
    number: 1,
    state: 'published',
    instructions: `You are a helpful assistant that...

1. Performs task A
2. Performs task B
3. Returns results`,
  }
]

Data Access Controls (dataAccess)​

MANDATORY when runAsUser is not set - Defines which roles the agent can inherit from the invoking user during execution.

interface DataAccessConfigType {
  roleList: (string | Role | Record<'sys_user_role'>)[]; // MANDATORY: List of role sys_ids or Role/Record references
  description?: string;                                   // Description of the data access configuration
}

Example:

dataAccess: {
  roleList: [
    '282bf1fac6112285017366cb5f867469',  // itil role sys_id
    'b05926fa0a0a0aa7000130023e0bde98'   // user role sys_id
  ],
  description: 'Role-based access for news collection',
}

When to Use:

  • Use dataAccess when you want the agent to run with dynamic user identity (inheriting roles from the invoking user)
  • Omit dataAccess when runAsUser is set to a specific user sys_id

Tools (tools)​

Tools allow agents to perform actions during execution. The type field acts as a discriminator — required fields differ per tool type.

Common Fields (all tool types)​

type BaseToolDetails = {
  name: string;                                                       // Tool name (MANDATORY)
  description?: string;                                               // Tool description
  active?: boolean;                                                   // Whether tool is active
  displayOutput?: boolean;                                            // Display tool output
  documentStatus?: 'pending' | 'indexing' | 'completed' | 'errored' | '';
  executionMode?: 'autopilot' | 'copilot';                            // Execution mode
  m2mSysOverrides?: string | Record<'sn_aia_agent_tool_m2m'>;        // Overrides for M2M record
  maxAutoExecutions?: number;                                         // Max automatic executions
  outputTransformationStrategy?: OutputTransformationStrategy;        // Output transformation strategy
  postMessage?: string;                                               // Message after execution
  postProcessingScript?: ((result: any, inputs: any, ...dependencies: any[]) => any) | string;
  preMessage?: string;                                                // Message before execution
  preRun?: boolean;                                                   // Run before other tools
  recordType?: 'custom' | 'promoted' | 'template';
  sysOverrides?: string | Record<'sn_aia_tool'>;                      // Overrides for tool record
  timeout?: string;                                                   // Timeout duration
  toolAttributes?: { [key: string]: JsonSerializable };               // Additional attributes
  transformationInstructions?: string;
  widgets?: (string | Record<'sn_aia_widget'>)[];
}

type OutputTransformationStrategy =
  | 'abstract_summary' | 'custom' | 'none'
  | 'paraphrase' | 'summary' | 'summary_for_search_results';

Input Field Definition​

interface ToolInputField {
  name: string;                    // Field name (MANDATORY)
  description: string;             // Field description (MANDATORY)
  mandatory: boolean;              // Whether field is required (MANDATORY)
  value?: string;                  // Default value
  invalidMessage?: string | null;  // Validation error message
  type?: FieldType;                // Field type
  mappedToColumn?: string;         // Column name for CRUD operations
  referenceConfig?: ReferenceConfig;
}

type FieldType =
  | 'boolean' | 'choice' | 'email' | 'glide_date_time' | 'html'
  | 'integer' | 'journal' | 'reference' | 'string' | 'translated_text'
  | 'url' | 'user_image';

interface ReferenceConfig {
  table: string | Table | Record<'sys_db_object'>; // Reference table
  field: string;                                    // Field to display
}

CRUD Tool Input Configuration​

interface ToolInputType {
  operationName: 'create' | 'update' | 'lookup' | 'delete'; // CRUD operation (MANDATORY)
  table: string | Table | Record<'sys_db_object'>;           // Target table (MANDATORY)
  inputFields: ToolInputField[];                             // Input fields (MANDATORY)
  queryCondition?: string;      // Query condition (e.g., 'email={{email}}')
  returnFields?: ToolReturnField[];
  limit?: number;               // Max records to return
  orderBy?: string;             // Field to order by
  sortType?: 'a_to_z' | 'z_to_a';
}

interface ToolReturnField {
  name: string;
  type?: FieldType;
  referenceConfig?: ReferenceConfig;
}

Trigger Configuration (triggerConfig)​

Configure automatic agent invocation based on record changes or schedules.

interface AiAgentTriggerConfigType {
  name: string;                   // Trigger name (MANDATORY)
  channel: string;                // Messaging channel (MANDATORY, defaults to Now Assist Panel)
  objectiveTemplate: string;      // Objective message template (MANDATORY)
  active?: boolean;               // Whether trigger is active (default: false)
  businessRule?: string | Record<'sys_script'> | ReturnType<typeof BusinessRule>;
  description?: string;
  domain?: string | 'global';     // Domain ID
  enableDiscovery?: boolean;      // Enable trigger discovery (default: false)
  m2mSysOverrides?: string | Record<'sn_aia_trigger_agent_usecase_m2m'>; // Overrides for trigger-agent-usecase M2M mapping
  notificationScript?: string | Record<'sys_script_client'> | ReturnType<typeof ClientScript>;
  portal?: string | Record<'now_assist_deployment_channel'>;
  profile?: string | Record<'now_assist_deployment'>;
  runAs?: string;                 // Field name for run-as user
  runAsScript?: ((context: unknown, ...dependencies: unknown[]) => string) | string;
  runAsUser?: string | Record<'sys_user'>;
  schedule?: ScheduleConfigType;  // Schedule for scheduled/daily/weekly/monthly triggers
  showNotifications?: boolean;    // Show notifications (default: false)
  sysOverrides?: string | Record<'sn_aia_trigger_configuration'>; // Overrides for trigger configuration record
  targetTable?: string;           // Table to monitor
  triggerCondition?: string;      // Condition script/query
  triggerFlow?: string | Record<'sys_hub_flow'>;
  triggerFlowDefinitionType?: TriggerFlowDefinitionType;
  usecase?: string | Record<'sn_aia_usecase'>;
  usecaseDiscovery?: boolean;     // Enable use case discovery
}

type TriggerFlowDefinitionType =
  | 'record_create' | 'record_create_or_update' | 'record_update'
  | 'email' | 'scheduled' | 'daily' | 'weekly' | 'monthly';

Schedule Configuration (schedule)​

Used when triggerFlowDefinitionType is 'scheduled', 'daily', 'weekly', or 'monthly'.

interface ScheduleConfigType {
  runDayOfMonth?: number;    // Day of month (1-31) — for monthly schedules
  runDayOfWeek?: number;     // Day of week (1=Sunday … 7=Saturday) — for weekly/daily
  repeatInterval?: string;   // Duration format: 'YYYY-MM-DD HH:MM:SS' (e.g., '1970-01-05 12:00:00' = every 5 days)
  starting?: string;         // Start date/time: 'YYYY-MM-DD HH:MM:SS'
  time?: string;             // Time of day: 'YYYY-MM-DD HH:MM:SS' (e.g., '1970-01-01 12:00:00' = 12:00 PM)
  triggerStrategy?: TriggerStrategy;
}

type TriggerStrategy = 'every' | 'once' | 'unique_changes' | 'always';

Example — Record-based trigger:

triggerConfig: [
  {
    name: 'News Collection Trigger',
    channel: 'now_assist_panel',
    objectiveTemplate: 'Collect latest news for topic: ${topic_name}',
    description: 'Triggers when high-priority topics are created',
    active: true,
    targetTable: 'x_my_app_news_topics',
    triggerFlowDefinitionType: 'record_create_or_update',
    triggerCondition: 'active=true^priority=high',
    showNotifications: true,
    enableDiscovery: false,
    domain: 'global',
  }
]

Example — Scheduled (daily) trigger:

triggerConfig: [
  {
    name: 'Daily Report Trigger',
    channel: 'now_assist_panel',
    objectiveTemplate: 'Generate daily report',
    active: true,
    triggerFlowDefinitionType: 'daily',
    schedule: {
      time: '1970-01-01 08:00:00',       // Run at 8:00 AM
      triggerStrategy: 'every',
    },
  }
]

Processes context before the agent runs. The script receives context as its first argument, plus any additional dependencies.

  return {
    pageContext: context.pageContext,
    triggerContext: {

Best Practices​

1. Security Configuration​

  • Always set acl to control who can access the agent
  • Use dataAccess.roleList when runAsUser is empty (dynamic user)
  • Set runAsUser to a specific user sys_id for fixed identity
  • Never use both runAsUser and dataAccess — choose one approach

2. Tool Configuration​

  • Always set type for all tools to specify the tool type
  • Use type-specific ID fields (e.g., capabilityId for capabilities, subflowId for subflows)
  • For script tools: Use a simple array of ToolInputField for inputs
  • For CRUD tools: Use a ToolInputType object for inputs with operation details
  • Use recordType: 'custom' for custom tools
  • Separate overrides: Use sysOverrides for the tool record, m2mSysOverrides for the M2M record
  • Input schema is auto-generated: Don't manually specify inputSchema — it's generated from inputs

3. Version Management​

  • Start with state: 'draft' for testing
  • Use state: 'published' for production
  • Increment number for new versions
  • Use meaningful name (e.g., "V1", "V2")

4. Trigger Configuration​

  • channel and objectiveTemplate are required fields on every trigger
  • Use specific triggerCondition to avoid unnecessary invocations
  • Set triggerStrategy: 'every' for all matching records
  • Set triggerStrategy: 'once' for one-time execution
  • For scheduled triggers use schedule (nested object) to configure time/interval/strategy
  • Use sysOverrides to override the trigger configuration record; use m2mSysOverrides to override the trigger-agent-usecase M2M mapping

5. Script Fields​

  • Use Now.include for external script files
  • Keep scripts concise and focused
  • Handle errors gracefully in scripts
  • Return proper status from tool scripts

Troubleshooting​

Tools Not Appearing in Agent​

Problem: Tools are generated as separate files but not linked to the agent.

Solution:

  • Ensure type is set for all tools
  • For script tools, use array of ToolInputField for inputs
  • For CRUD tools, use ToolInputType object for inputs
  • Check that type-specific ID fields are set correctly (e.g., capabilityId, subflowId)

Data Access Validation Error​

Problem: Error: "dataAccess.roleList is mandatory when runAsUser is not set"

Solution:

  • Either set runAsUser to a user sys_id
  • Or provide dataAccess.roleList with at least one role

Trigger Not Firing​

Problem: Trigger configured but agent not invoked automatically.

Solution:

  • Verify triggerConfig.active is true
  • Ensure channel and objectiveTemplate are set (both required)
  • Check triggerCondition matches your records
  • Ensure targetTable is correct
  • For scheduled triggers, verify the schedule object is correctly configured

Script Fields Not Working​

Problem: Scripts showing as [object Object] or not executing.

Solution:

  • Use string format for scripts: `(function() { ... })()`
  • Or use function format: (inputs) => { ... }
  • Ensure CDATA tags are NOT manually added (handled automatically)
